# Public EarthScope Voice Reference Set

This document captures the strongest and weakest recent public EarthScope post examples so the writer can be tuned against real approved taste instead of abstract style language.

It is intended to support:

- shadow-mode review
- prompt revision
- founder-voice preservation
- title and caption evaluation

- `Supabase Snippet Weekly Default Posts Fetch.csv`
  - February 7, 2026 through February 14, 2026
- `Supabase Snippet Weekly Default Posts Fetch-2.csv`

## Core Finding

The stronger posts do not win because they are more dramatic.

They win because they feel more authored.

They usually do four things well:

1. describe the day in a fast, human way
2. use one image or analogy that actually fits
3. turn quickly into pacing advice
4. keep the humor light and relevant

The weaker posts usually fail by sounding like a generated system explanation with a metaphor dropped on top.

## Extracted Voice Rules

### What the writer should do

1. Start with a human take on the day, not a condition label.
2. Use at most one analogy in a caption.
3. Make the analogy do real work for pacing or expectation-setting.
4. Move quickly from day texture to practical advice.
5. Let the caption feel like a person reacting to the day, not a forecast desk summarizing it.
6. Keep humor light, situational, and relevant.
7. Let the metrics footer carry the hard numerical texture so the caption itself can stay more human.

### What the writer should avoid

1. Do not open with bulletin phrasing like `Geomagnetic activity is...`
2. Do not use a metaphor just because the system wants one.
3. Do not use stock titles unless there is no better option.
4. Do not sound like the app’s explainer voice.
5. Do not stack several pieces of technical context before giving the user the point.
6. Do not make the caption read like a cleaned-up model output.

## Title Rules

Good public titles should feel:

- specific
- slightly odd in a memorable way
- short
- human

They should not feel:

- category-like
- dashboard-like
- reused across many days

Preferred title behavior:

- one image
- one motion
- one mood

Avoid as defaults:

- `Steady Field`
- `Magnetic Calm`
- `Active Geomagnetics`
- `Geomagnetic Storm Watch`
- `Clear Runway`

These may still be acceptable as fallbacks, but they should not dominate.

## Caption Rules

Recommended caption structure:

1. day texture line
2. one implication line
3. one pacing or support line
4. optional lightly funny or memorable turn

Not recommended:

1. field conditions summary
2. more field conditions summary
3. one analogy
4. practical advice

## Body Rules

The body does not need to sound as playful as the caption.

It should be:

- readable
- grounded
- practical

But it should still avoid:

- interchangeable boilerplate
- overuse of system phrasing
- too much repeated cardio/HRV language unless the day really calls for it

## Review Rubric For Shadow Mode

Use this rubric when comparing live versus candidate output.

### Approve if

- it sounds more like an authored founder voice
- it uses one clear image instead of several weak ones
- it gets to the point faster
- it keeps the same factual meaning
- the humor feels relevant to that day

### Reject if

- it sounds more like a report
- it sounds more like app copy
- the joke feels generic
- the title could fit ten other days equally well
- the analogy feels inserted rather than earned

## Immediate Do / Do Not Set
